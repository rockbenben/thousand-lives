# 航海「怒海争锋」机缘体系 设计（spec）

> 第 6 个题材。**核心立意：白手起家、纵横七海的个人霸业**——船级前进式（只进不退），**财富为魂**（财富是扩张引擎：赚钱→买船养人→升势力）。区别于三国(volatility)与科幻(集体求生)。

## 现状（voyage.ts，~980 行）
- 3 属性：`ship 船力`(init35, deathBelow0, **decay1**, 主死亡线；bands 即船级：漏水残船/老旧帆船/坚船利炮/一方雄舰/无敌舰队)、`wealth 财富`(init25, 非死亡)、`crew 人心`(init60, deathBelow0, 哗变第二死亡线)。
- 3 开局：破产商人之子 / 哗变水手 / 落魄贵族航海家——**均无 flag**。
- **无** ceilingUnlocks、tierLabel、flagsSet/flagsClear/endTone。
- ~80 个 localEvents（含 keyMoment：海上肥羊/飓风夺命/心腹背叛/舰队封锁/王室招安/决战在即/旧仇可雪；itemsGained 因果链：祖传海图/古舰巨炮/青铜罗盘→完整罗盘→藏宝岛坐标→黄金宝藏、海军通行勋章→军港换舰、海妖残页→呜咽礁群、宿敌悬赏令→循令端宿敌）。
- ~27 个结局，多为 maxTurns 门控的 ship/wealth/crew 组合 + ship<=0/crew<=0 死亡。

## 设计总览（船级前进式 + 财富为魂）

### 1. 进阶脊梁：海上势力阶梯（挂 `船力` ceilingUnlocks，只进不退）
```
ship: { initial:35, max:100, deathBelow:0, decayPerTurn:1,
  ceiling: 35,                      // base = initial（三国同款：服务无名小卒的船长，船力卡在老旧帆船档，唯立威方能壮大）
  ceilingUnlocks: [
    { flag:'私掠', max:50 },         // 站稳脚跟：第一桶横财、扬名立威
    { flag:'船队', max:70 },         // 拉起一支船队/结盟成势
    { flag:'海枭', max:88 },         // 破局壮大、威震一方海域
    { flag:'霸主', max:100 },        // 称霸七海（解锁「无敌舰队」band）
  ] }
```
- tierLabel = **势力**。AI【晋阶之序】叙述「本剧势力依次为：私掠→船队→海枭→霸主」。
- **只进不退**（无 flagsClear；区别于三国 volatility）。怒海的凶险由**双死亡线**承担：沉船(船力)+哗变(人心)。
- base35 = init：升 私掠 前船力卡 35（老旧帆船），早期船力增益被 clampEffects 压在 35——「一艘抵债旧船，扬名前难壮大」（同三国 wit 卡 initial）。

### 2. 财富为魂（本题材独特点）
升势力闸门需**递增的财富门槛**（扩张要花钱买船募人）：私掠(无门槛，靠劫掠起家)→船队(wealth>=40)→海枭(wealth>=55)→霸主(wealth>=70)。财富＝扩张引擎，可被掠夺、可挥霍。三资源转换循环(赚钱→船力+人心)，是航海区别于其它题材的脊。

### 3. 四道升势力闸门（keyMoment + flagsSet，改造既有，has(prev) 串链）
授印记的是**「壮大/立威」那一支**；另一支只渡过、不晋阶。

| 印记 | 改造既有事件 | 新增门控 `requires` | 授印记的选项 |
|---|---|---|---|
| **私掠** | `海上肥羊`(keyMoment,weight1.2,早) | （无；first gate）| 「升起黑旗，登船劫掠」→ flagsSet:['私掠'] |
| **船队** | `海盗结盟`(minTurn8) | `has(私掠) & wealth>=40`（+keyMoment）| 「歃血结盟，借势纵横怒海」→ flagsSet:['船队'] |
| **海枭** | `老提督甲板献策`(minTurn10，本就讲「舰队撞高墙、破而后立」)| `has(船队) & wealth>=55`（+keyMoment，替原 ship>=45）| 「主动挑战列强舰队，破而后立」→ flagsSet:['海枭'] |
| **霸主** | `决战在即`(keyMoment,weight1.2) | `has(海枭) & wealth>=70`（替原 ship>=65）| 「设伏火攻，险中求一场大胜」→ flagsSet:['霸主'] |

- 同回合即时生效（引擎 applyChoice 先应用 flags 再 clamp）。
- has(prev) 串链保证顺序；财富门槛使「不富则难升级」。
- **sim 杠杆**：若某闸门晋阶支被策略极少选中致登顶≈0，则改两支都授印记。

### 4. 三开局身份印记 + 三身份事件（新增，不删既有）
| 开局 | flag | 新增身份事件主题 |
|---|---|---|
| 破产商人之子 | `商人之子` | 精于行情海贸——一桩唯独识货者能把握的市场机变（巧取暴利 vs 守信立名）|
| 哗变水手 | `哗变水手` | 通缉恶名缠身——官府悬赏/旧日兄弟义气的两难（亡命到底 vs 顾全弟兄）|
| 落魄贵族航海家 | `贵族航海家` | 旧贵族身份执念——昔日同侪/祖业认人的尴尬（攀附旧圈 vs 决绝向海）|

### 5. 隐藏 endTone 哨兵（`crew<=-1`，2 地狱 + 1 天堂；靠既有抉择加权 outcomes[].endTone 强制）
新增 3 结局 `condition:'crew<=-1'`（人心 floors 0，永不自然触发；仅 endTone 强制）。沿科幻/三国 outcomes 模式。

| 基调 | 性质 | 宿主事件·支 | 权重(survive:哨兵) |
|---|---|---|---|
| **屠岛劫财·恶贯满盈** | 地狱 | `孤岛通商`「恃强凌弱，巧取豪夺」| 1:1 |
| **见利忘义·众叛弃尸** | 地狱 | `藏宝图争夺`「独闯死亡海域，富贵险中求」(弃弟兄于死地) | 1:1 |
| **逍遥怒海·自由之王** | 天堂 | `王室招安`「婉拒王命，宁做自由的怒海之主」(于巅峰拒冠而逍遥) | 4:1（稀有）|

- 哨兵均用 `crew`（人心向背是航海之魂）。测试：每个哨兵基调被某事件 endTone 引用（防 tone 打错）。

### 6. apex 门控（无需改结局——ceiling 自动达成）
- 船级印记锁船力 ceiling，既有高船力结局**自动**隐式需走完势力：海上霸主·七海之王(ship>=88)⟺有海枭；受封提督(ship>=85)⟺有海枭；怒海枭雄(ship>=80)⟺有船队。霸主印记解锁「无敌舰队」band(船力89-100)。
- **财富路保留**：海贸巨擘·富甲七海(wealth>=80&crew>=70)、富可敌国(wealth>=85&ship>=45) 不靠高船级即可达——多路径并存，符合「财富为魂」。
- 只进不退，船力高 ⟺ 已抵该印记，**无需在结局加 has() 兜底**（同科幻）。保持结局现状。

### 7. AI 注入 & systemPrompt
- 加 `tierLabel:'势力'` → prompt.ts【晋阶之序】自动用本剧术语 + 势力印记序（prompt.ts 已通用化，不动）。
- systemPrompt 补：势力分私掠→船队→海枭→霸主、唯重大机缘（立威/壮大的关键抉择）方能晋阶、财富是扩张引擎；泯灭道义的僭越（屠掠/弃友）或拒冠逍遥的极难抉择可触发隐藏结局。**不讲数值/封顶**。

### 8. sim-balance
跑 `npx vite-node scripts/sim-balance.ts voyage 5000`。校准：死亡率健康（沉船+哗变+endTone 致死）；登顶（海枭/霸主+高 stats）random 稀有但 survive/greedy 可达；据此定 `人心` decay（起 0，若众叛死<3% 则加 1）。

## 不做（YAGNI）
- 不加船级 flagsClear/倒退（前进式；volatility 是三国本质）。
- 不重写既有 ~80 事件/~27 结局文案；只做最小机缘叠加。
- 不动 maxTurns(30)。

## 实现顺序（plan ~6 任务）
1. 船力势力 ceilingUnlocks（+测）→ 2. 三开局 flag + 三身份事件（+测）→ 3. 四升势力闸门改造串链（财富门槛）（+测）→ 4. 三隐藏 endTone 哨兵（+测）→ 5. tierLabel + systemPrompt + AI 注入测 → 6. 人心 decay 与 sim 校准。

## ★复用教训（科幻期踩的坑，本题材务必避免）
带 `outcomes` 的**选项本身**必须写 `effects: {}`（local.ts magOf 读 c.effects，缺则 sim 崩）；每个 outcome 分支（含纯 endTone 分支）也必须有 `effects: {}`（Outcome 类型 effects 必填，缺则 **tsc TS2741**，vitest 不报）；endTone 分支补 `reaction`（结局前叙事拍点）。**收尾必跑 `npx tsc --noEmit`**，别只信 vitest。
