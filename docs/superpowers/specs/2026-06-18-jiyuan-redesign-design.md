# 千世书 · 机缘重构设计（xian 旗舰）

日期：2026-06-18
状态：设计已确认（§1-5），待写实现计划

## 1. 背景与问题

当前游戏的根本缺陷：玩法是「管理 3 个数字、活到满期 → 飞升几乎必然」。后果——

- **苟命即最优解**：属性短期加减无所谓，只要活着，修为迟早登顶、结局必好。
- **道心/侠名等属性形同虚设**：不真正门控任何东西。
- **缺乏戏剧张力**：没有「机缘错过就一辈子卡关」的稀缺感，也就没有修仙小说里打打杀杀的根本动机。
- **后果太确定**：选项 = 固定 effects，只有「命运无常」给个 ±缩放；没有因果、没有「碰上就天堂/地狱」的无常。

本设计以 **xian（缥缈仙途）为旗舰**，走通 L0（世界框架）→ L1（引擎能力）→ L2（剧本内容）一条龙，作为模板，之后再把 L0/L2 推广到其他剧本。L1 引擎只做一次、全剧本 + 双模式（本地 / AI）共享。

已落地并保留的前置工作：每回合衰减 `decayPerTurn`（寿元等「流逝时钟」），是本重构「逆水行舟」张力的一块。

## 2. 目标与非目标

**目标**
- 把「境界突破」从「数字到了就升」改为「抓住稀缺机缘」；错过/失败 → 修为封顶、可能一辈子卡关 → 走向「没突破」的软结局。
- 让道心等属性真正门控进阶（道心低 → 抓不住机缘 → 走火入魔）。
- 选择后果可随机（善报/恶报/徒劳）且有延迟因果。
- 身份（魔道/仙门/草根）决定可遇事件与支线。
- 提高随机/意外事件占比；增加主角暴毙几率。
- 隐藏事件「无视数值，碰到就天堂或地狱」。

**非目标（留待后续 spec）**
- 把 L2 内容推广到其他 8 个剧本（本 spec 只做 xian）。
- 三国 sanguo 的「早期速通」深度重构（独立问题）。
- 重做 UI / 存档格式大改（新字段一律可选，旧存档兼容）。

## 3. 统一原语：印记（flags）

一套具名隐藏状态（`state.flags: string[]`），由事件/选择写入，可被三处读取：① 事件门控 ② 结局条件 ③ 属性封顶。一个原语点亮六诉求中的五个：

| 诉求 | 用印记实现 |
|---|---|
| 机缘/突破 | 「金丹」印记把修为天花板 45→70；无印记则修为再练也卡筑基 |
| 道心有用 | 突破机缘事件 `requires 道心≥55`；道心低 → 抓不住 → 走火入魔 |
| 身份事件 | 开局写入「魔道」印记，魔教专属事件 `requires has(魔道)` |
| 因果种子 | 「救老人」分支埋隐藏印记，数载后随机触发善报/恶报/徒劳事件 |
| 隐藏天堂地狱 | 事件分支携带 `endTone`，无视数值当场定结局 |
| 软结局 | 结局条件读印记：`maxTurns & !has(金丹)` → 「蹉跎一世·止步筑基」 |

复用现有 inventory/requires 机制（物品已是类印记的字符串集合），把「可读取面」扩展到结局条件与属性封顶。印记与物品的区别：物品对玩家可见、印记是隐藏机制。

## 4. 机缘 / 境界封顶（keystone）

- 修为 `cultivation` 仍是 0-100 带 bands，但加**有效上限**：初始天花板 45（筑基顶），由印记逐级解锁。
- 突破机缘 = 在窗口出现的稀有事件，门控 + 掷骰：
  - 例「凝结金丹」：窗口第 8-15 载，`requires cultivation>=40 & daoHeart>=55`，低权重（稀有）。
  - 抓住且成功 → 得「金丹」印记，天花板升 70；失败 → 重伤（寿元-）甚至走火（道心-）。
  - 错过窗口 → 金丹机缘几十载一遇或再不出现 → 大概率卡死筑基 → 软结局。
- 苟命不再够用：要飞升必须主动争机缘，而争机缘要冒险。

## 5. 因果 & 随机后果

**即时分支 `outcomes`**：选项可声明一组加权结局，引擎掷骰取一，每分支带自己的 effects/旁白/印记/可选 `endTone`：
```
帮路边老人 → 60% 徒劳无功（小损时间）
             25% 恶报：仇家寻上门（寿元/性命-，埋「结怨」印记）
             15% 善报：隐世高人传承（得印记/大益）
```
有 `outcomes` 走掷骰；否则走旧 `effects`（向后兼容）。**规则**：带 `outcomes` 的选项**跳过命运无常**（分支掷骰本身已是随机，避免双重随机）；只有走旧 `effects` 的选项才吃命运无常的 ±twist。

**因果种子（延迟）**：某分支埋隐藏印记（「结怨·仇家」），数载后事件 `requires has(结怨)` 才触发回收（又可好/坏随机）。何时炸、炸成什么样，皆涌现，玩家无法预读。

**事件频率**：(a) 更多事件改用 `outcomes` 携带变数；(b) 调高 `FORTUNE_CHANCE`；(c) 加低权重「意外/奇遇」野事件（`wildcard`），任何回合可乱入并有保底权重。

## 6. 隐藏事件 / 暴毙 / 天堂地狱

- **隐藏事件**：稀有（低权重，常由印记/手气门控），分支可携带 `endTone` **无视当前属性直接定结局**：
  - 地狱：误入上古杀阵 / 撞天劫 / 踩中凶煞 → 当场暴毙，再健康也没用。
  - 天堂：遇仙人传承 / 得泼天造化 → 直接拿高阶印记（跳过一次突破）甚至飞升捷径。
- **暴毙几率**：隐藏地狱 + §5 恶报死亡，抬起暴毙率，做出「天威难测、福祸无常」。关键是**稀有**（碰上才生效，不是天天死），且**冒险拉高概率**——探秘境/夺宝同时拉高天堂与地狱的骰子。

## 7. 引擎架构（L1，schema/engine 改动）

| 能力 | schema 改动 | engine 改动 |
|---|---|---|
| 印记 flags | `GameState.flags: string[]`；choice/outcome 加 `flagsSet/flagsClear` | 写入 flags |
| 条件语言扩展 | — | `parseCondition` 支持 `has(NAME)` / `!has(NAME)`；`evalCondition` 接收 flags。事件门控 + 结局共用 |
| 机缘封顶 | attribute 加 `ceiling?: number` + `ceilingUnlocks?: {flag,max}[]` | `clampEffects` 用「印记解锁后的有效上限」截断 |
| 加权分支 outcomes | choice 加 `outcomes?: {weight,effects,reaction?,narrative?,flagsSet?,flagsClear?,itemsGained?,itemsLost?,endTone?}[]` | 有 outcomes 则按权重掷骰取一，否则走旧 effects |
| 强制结局 forceEnding | outcome/event 加 `endTone?: string` | 命中即 `ended={tone}`，绕过数值；epilogue 仍从 `sc.endings` 按 tone 取 |
| 身份印记 | opening 加 `flag?: string` | `initState` 写入 `state.flags` |
| 频率 | event 加 `wildcard?: boolean`；调 `FORTUNE_CHANCE` | `pickLocalEvent` 给野事件保底权重 |

**触点文件**：`src/engine/types.ts`、`src/scenarios/schema.ts`、`src/engine/condition.ts`、`src/engine/state.ts`、`src/engine/local.ts`（pickLocalEvent/localTurn）。

**AI 模式同步（两者共享引擎）**：AI 回合返回 schema（`src/ai/turn.ts` / `types.ts` 的 TurnResult）加 `flagsSet/endTone`；systemPrompt 指导 GM 何时授印记、触发隐藏结局、尊重境界封顶；引擎对**封顶与强制结局兜底强制执行**，不全靠 AI 自觉。

**向后兼容**：所有新字段可选；旧存档无 `flags` → 默认 `[]`；无 `ceiling` → 等同 `max`；无 `outcomes` → 走 `effects`。

## 8. 世界框架 + 身份（xian 的 L0/L2）

**世界轴**：xian 定为 **末法 / 灵气衰微**——资源稀少、突破凶险、飞升者百年难见。稀缺逼出争夺与打杀。写进 systemPrompt，并据此校准机缘事件的权重/窗口（机缘要「稀」）。

**身份支线**（开局写身份印记，解锁对味事件弧）：
- 魔道余孽 → 夺舍/屠戮/采补：修为暴涨但崩道心、引正道追杀（快而险）。
- 仙门弟子 → 宗门任务/门规/长辈庇荫与倾轧（稳但受束缚）。
- 草根散修 → 资源极匮、处处碰壁，开局带「无门无派」印记（机缘更稀但自由）。

## 9. 成功标准（可量化 + 体感）

用 `scripts/sim-balance.ts`（扩展以追踪印记/突破/强制结局）验证 xian：
- 「苟命」(survive) 策略**不再近乎必然飞升**：飞升率降到稀有；「没突破」软结局成为被动玩法的常见归宿。
- **道心门控生效**：低道心局无法金丹 → 卡关。
- **暴毙率**：隐藏地狱带来一个「稀有但真实」的暴毙率（数字明面也防不住）。
- **身份分流**：不同开局命中不同事件集（魔道局明显多「屠戮/追杀」事件）。
- 体感：playtest（browse）确认「争机缘」的紧张与「福祸无常」的味道。

## 10. 范围与顺序

本 spec = **xian 旗舰**：
1. **L1 引擎**（§7，全剧本+双模式共享）——先做，TDD，每个原语配单测（参照 `decayPerTurn` 的做法）。
2. **L0 xian 世界框架**（§8）——systemPrompt 世界设定 + 属性含义。
3. **L2 xian 内容**——加封顶、突破机缘窗口、身份弧、因果种子事件、隐藏天堂地狱事件；重写/扩展 `xian.ts` 的 localEvents + systemPrompt。

**后续独立 spec**：L0/L2 推广到其他 8 剧本；sanguo 深度重构。

## 11. 风险

- **内容量**：xian L2 事件量大（当前已 1600 行），新增机缘/身份/因果/隐藏事件会更多——分批写、用 sim 校准。
- **暴毙平衡**：隐藏地狱太频会劝退；必须稀有且「冒险才触发」。
- **AI 遵从**：AI 可能不按印记/封顶出牌 → 引擎兜底强制执行（clamp + forceEnding）。
- **存档兼容**：新字段全可选、给默认值，旧存档不破。

## 12. 测试策略

- 每个引擎原语 TDD 单测（flags 读写、条件 `has()`、ceiling 截断、outcomes 掷骰、forceEnding、opening flag）。
- 扩展 `sim-balance.ts`：统计飞升率、各境界卡关率、暴毙率、身份事件命中分布。
- `browse` 手动 playtest 三种身份各一局，确认体感。
- 全量 `vitest` + `tsc` 绿。
