# 千世书 · L2b：xian 深度内容（身份弧 / 因果种子 / 隐藏天堂地狱）

日期：2026-06-18
状态：设计已确认（三支柱 + sim 守门），待写实现计划
前置：L1 引擎、L2a（xian 机缘重构）均已合入 main。父设计 `2026-06-18-jiyuan-redesign-design.md`、`2026-06-18-l2a-xian-config-design.md`。

## 1. 目标

给 xian 的事件池注入深度与涌现性，让重玩有别、选择有因果、世界有「碰上就天堂或地狱」的无常。**全部为 `xian.ts` 的内容(localEvents/endings)增补，复用 L1 原语(flags/has()/outcomes/endTone)，不改引擎。**

**非目标（后续）**
- L2c：AI 回合 schema(flagsSet/endTone) + systemPrompt 世界观注入。
- 其他 8 题材的同类改造。
- 新引擎能力（本期不加）。

**硬约束**
- **保住 L2a 平衡**（sim 守门，详见 §5）：技术流登顶 ~个位数%、**无 <10 回合收场**、死亡有真实占比、乱点多止步低境界。
- 致死/暴毙类隐藏事件一律 `minTurn>=10`；早期负向单事件 effect 不致 10 回合内归零。
- 不必兼容旧存档（结构已在 L2a bump，沿用）。

## 2. 支柱① 身份支线弧

**起点分化（补 L2a 缺口，引擎无关）**：L1 schema 不支持按开局给不同初始属性，故用**第 1 回合保证触发的一次性身份事件**等价实现。每身份一个 `requires has(身份)`、`minTurn:1, maxTurn:1, once:true` 的事件，其唯一/主分支用 effects 拉开起点：
- 魔道 `flag:魔道` → `{cultivation:+6, daoHeart:-15}`（魔功起步快、道心先天有亏）。
- 仙门 `flag:仙门` → `{daoHeart:+8}` + `itemsGained:['宗门信物']`（资粮庇佑）。
- 散修 `flag:散修` → `{cultivation:-2}`（家底全无，但无门规束缚）。

**专属弧事件**（各身份 4-6 个，`requires has(身份)`，部分 `once`）：
- **魔道**：屠戮/采补类——修为暴涨但道心崩，且分支埋「正道追缉」种子(见 §3 配合)；被追杀事件 `requires has(正道追缉)`。
- **仙门**：宗门任务给资源/物品，违门规则受惩戒；长辈庇荫与宗门倾轧。
- **散修**：资源匮乏的碰壁与孤身奇遇，机缘更稀但选择更自由。

## 3. 支柱② 因果种子

用 outcomes 埋隐藏印记 + 延迟 `requires` 回收：
- **种子事件**：某选项的某 outcome 分支 `flagsSet:['结怨·X']`（或善缘/收徒等），当下只有小效果。
- **回收事件**：`requires has('结怨·X')`、`once:true`，命中时 `flagsClear:['结怨·X']`；choices/outcomes 随机**善报/恶报/徒劳**三态。延迟感来自「种子中途才种下 + 回收事件要被抽到」的涌现，无需额外计时。
- 约 4-5 组种子，例：
  - 帮路边老人 →（种子 善缘/结怨）→ 仇家寻仇(寿元/道心-) / 隐世高人传承(印记或大益) / 徒劳无功。
  - 放过仇敌 → 反噬(被反杀险) / 感化(得助力)。
  - 收徒传艺 → 叛徒噬主 / 衣钵有传(益)。

## 4. 支柱③ 隐藏天堂地狱

稀有(低 weight)、`minTurn>=10`、`endTone` 无视数值定结局：
- **地狱**：误入上古杀阵 / 踩中凶煞 / 提前撞天劫 → 某 outcome 带 `endTone` 当场暴毙。选项里「探/夺」会拉高遇险分支权重(冒险才触发)。
- **天堂**：遇仙人传承 / 得泼天造化 → 要么 `flagsSet` **直接跳一境界印记**(大造化，配寿元回补)，要么**极稀有** `endTone` jackpot 飞升。
- **新增结局基调 + epilogue**（在 `xian.endings` 增条目，供 endTone 与图鉴）：
  - 横死类：`误入杀阵·横死当场`、`凶煞缠身·暴毙荒野`。
  - 登天类：`仙缘垂青·一步登天`（jackpot）。
  这些结局**仅由 endTone 命中**。schema 要求每个 ending 有 condition，故统一用**永不自然成立**的占位 `lifespan<=-1`（lifespan 被 clamp 在 [0,max]，恒不满足），确保只能经 endTone 驱动、绝不被数值误触。

## 5. 平衡守门（sim）

每加一支柱后 `npx vite-node scripts/sim-balance.ts xian 3000`，断言 L2a 基线仍成立：
- 技术流(survive)**登顶率 ~个位数%**（隐藏天堂可把上限略抬到 ≤~12%，但 jackpot 飞升须极稀有，单独占比 < ~1%）。
- **P(收场回合 < 10) == 0**（三策略）。
- **死亡有真实占比**（非 0），且不暴涨到劝退（random 死亡 ≤ ~50%）。
- 乱点(random)仍多止步炼气/筑基。
任何支柱把某指标推出范围，调该支柱事件的 weight/minTurn/effects，重跑直至回归。

## 6. 触点文件

- `src/scenarios/xian.ts`：localEvents 增补三支柱事件；endings 增几个隐藏结局基调。
- `src/scenarios/xian.test.ts`：守护与功能测试(身份起点事件命中、种子→回收链、隐藏 endTone 定结局、登顶率不破)。
- `scripts/sim-balance.ts`：如需新度量(如各身份分布/种子触发率)可加，否则复用。

## 7. 成功标准

- 三身份开局走出可辨差异的早期轨迹(sim 或试玩可见)。
- 因果种子链可端到端触发(测试构造 has(种子)→回收三态)。
- 隐藏天堂地狱能 endTone 定结局且稀有(sim 占比在 §5 带内)。
- L2a 平衡全保住(§5)。
- 全量 `vitest` + `tsc` 绿。

## 8. 风险

- 内容量大、且新事件改变事件池权重 → 每支柱 sim 守门，分批加。
- 隐藏天堂(跳境界/ jackpot 飞升)易破坏登顶稀有度 → weight 压到极低，sim 盯 jackpot 单独占比。
- 身份起点用「第 1 回合事件」而非初始属性：须确保该事件**必触发**(minTurn=maxTurn=1 + once + has(身份))且不被其它 turn-1 事件挤掉——实现时验证三身份开局必命中各自起点事件。
- 隐藏结局的 condition 占位若写不当可能自然误触 → 用不可达条件 + 仅靠 endTone。

## 9. 测试策略

- 身份起点：三身份 initState 后跑一回合，断言命中各自起点事件、属性按预期偏移。
- 因果：构造 has(种子) 状态，断言回收事件可被 pick 且三态 outcomes 生效、回收后 flagsClear。
- 隐藏：构造取到 endTone 分支，断言 `st.ended.tone` 为对应隐藏基调、无视属性。
- sim 守门断言(§5)。
- 守护：xian 仍过 schema、所有结局条件可解析、隐藏结局不被数值自然误触。
