# 废土「末世求生→末世重建」机缘体系 设计（spec）

> 第 7 个题材，**最大的一次改造**。废土是首个纯生存题材，原「撑 30 天等军方救援」的短期围城框架容不下机缘体系的成长弧，也与它自己的大野心（建立势力/重建文明）打架。**用户拍板方向 A**：扩成长期末世（turnUnit 天→月、maxTurns 30→36 三年），前提从「等救援」改为「**救援未至、安全区接连失守，你要在废土上长期求生并亲手重建一方据点与秩序**」。脊梁＝**据点前进式**（只进不退），是黑夜里的希望弧，与生命/理智的衰减压力成对照。

## 现状（wasteland.ts，~1600 行）
- 3 属性：`hp 生命`(init80, deathBelow0, **decay1**, 死亡线)、`sanity 理智`(init70, deathBelow0, 疯癫死亡线、**无 decay**)、`supplies 物资`(init50, 非死亡、每回合靠事件 effects 消耗 -3~-10)。
- 3 开局：便利店店员 / 退役军医 / 高中生——**均无 flag**。
- **无** ceilingUnlocks/tierLabel/flags/flagsClear/endTone。turnUnit **天**，maxTurns **30**。
- ~80 个 localEvents（求生危机/道德两难/奇遇；keyMoment：空投补给/伙伴感染/检查站登记/免疫之子/死亡列车/尸潮围城/末日食人；**无建据点里程碑事件**）。
- **23 个结局**：2 死亡（hp<=0 力竭身亡 / sanity<=0 疯癫失智）+ 21 个 `maxTurns & <stats>` 变体，**全部以「第30天军方救援抵达/获救/撤离」为框架**。

## 设计总览

### 1. 时间尺度 + 前提重构（官场式长度扩展的加强版）
- **turnUnit 天→月；maxTurns 30→36**（三年）。
- **intro 重写**：从「撑过三十天等军方救援」→「不明病毒爆发后城市陷落，许诺的军方救援迟迟未至、官方安全区接连失守。三年，你要在崩坏的废土上活下去，并亲手从废墟里重建一方据点与秩序」。
- **systemPrompt 重写**：每回合＝一个月（剧情按月推进，可跨度更大的事件/经营）；删「临近第30天铺垫军方救援」，改「救援已成泡影，玩家须长期求生并经营据点；物资每回合消耗规则不变」。
- **救援框架的中段事件轻改**：`检查站登记`(minTurn24,weight2)等「抵达安全区」事件轻度重构为「一处仍在勉力支撑/终将失守的安全区」，不再暗示终局获救（其余求生事件作为「某月的遭遇」原样可用）。

### 2. 进阶脊梁：据点阶梯（挂 `物资` ceiling，只进不退）
孤身流浪只能手到口（物资封顶 50=init）；**建据点才能屯积更多、缓冲生命/理智**。
```
supplies: { initial:50, max:100,
  ceiling: 50,                      // base = initial（无据点则物资卡 50，囤不住）
  ceilingUnlocks: [
    { flag:'落脚点', max:65 },        // 觅得并清空一处可守的藏身点
    { flag:'据点', max:80 },          // 加固成真正的基地（储水储粮/陷阱）
    { flag:'堡垒', max:92 },          // 筑墙设防、可挡尸潮
    { flag:'营地', max:100 },         // 幸存者来投、号令一方聚落
  ] }
```
- tierLabel = **据点**。AI【晋阶之序】叙述「本剧据点依次为：落脚点→据点→堡垒→营地」。
- **只进不退**（无 flagsClear；废土已有 hp/理智 双衰减作「崩坏」压力，据点是对照的希望弧，不再叠 volatility）。
- base50=init：建落脚点前物资卡 50，早期大笔物资被 clampEffects 压在 50——「孤身一人屯不住，只够手到口」。

### 3. 四道建据点闸门（keyMoment + flagsSet，**全新增**，has(prev) 串链）
现有事件全是求生危机、无建据点里程碑，故 4 道闸门**新增**。授印记的「建造」支耗物资/冒险（投入换长远），另一支只苟且、不晋阶。
| 印记 | 新增闸门主题 | `requires` |
|---|---|---|
| **落脚点** | 寻得一处可守之地、清场据为巢穴 | （无；first gate，早期 minTurn~2）|
| **据点** | 加固扩建（储粮储水、设陷阱）| `has(落脚点)` |
| **堡垒** | 筑墙设防、扛住一次尸潮试探 | `has(据点)` |
| **营地** | 接纳投奔的幸存者、立下规矩号令一方 | `has(堡垒)` |
- 同回合即时生效（applyChoice 先 flags 再 clamp）。建造支带 supplies 负增益（投入）但 flagsSet 印记。
- **sim 杠杆**：若闸门晋阶支被策略极少选中致登顶≈0，改两支都授印记。

### 4. apex 靠 ceiling 自动门控（结局条件基本保留）
据点印记锁物资 ceiling，故高物资结局**自动**隐式需建到对应据点：
- `末世王者·重塑秩序`(supplies>=80)、`重建据点·重燃文明`(supplies>=80) ⟺ 有 据点（ceiling80）；`据点之主·守得云开`(supplies>=70) ⟺ 有 据点；`满载而归`(supplies>=70) ⟺ 有据点。
- 不建据点者物资封顶 50，只能落 hp/理智 向的结局（钢筋铁骨/心如磐石/苟延残喘/熬过末世 兜底）——「不经营据点＝得个人求生类结局」，符合机缘立意。营地印记解锁满屯(93-100)。

### 5. 结局重构（救援框架→三年末世归宿）
23 个结局**条件基本保留**（hp/理智/物资 仍是对的轴），**重写 epilogue 框架**：去「军方救援抵达/军旗/担架/获救」，改「三年了，你在废土上的归宿」——高物资+身心→建成据点/重燃文明（废土灯塔）；纯硬朗→钢筋铁骨熬住；纯清醒→心如磐石；低→苟延残喘/形神俱疲/油尽灯枯；死亡 hp<=0/sanity<=0 保留。**含「获救/撤离」的 tone 名一并改**（如 从容获救→从容立足、安稳撤离的幸存者→安稳扎根的幸存者、获救→熬过末世）。plan 给前提文案 + 改名表 + 3~4 个范例重写 + 规则，余下按范式重写（内容任务，sonnet + 评审）。

### 6. 三开局身份印记 + 三身份事件（新增）
| 开局 | flag | 新增身份事件主题 |
|---|---|---|
| 便利店店员 | `店员` | 熟悉物资/街区——一处只有老店员才摸得清的隐藏库存（独吞 vs 分享聚人）|
| 退役军医 | `军医` | 急救本领被需要——重伤者求救（耗己物资药品救人 vs 自保）|
| 高中生 | `高中生` | 年轻冲动/体力好——逞勇涉险的诱惑（莽撞搏一把 vs 学会克制）|

### 7. 隐藏 endTone 哨兵（`sanity<=-1`，2 地狱 + 1 天堂；人性崩坏挂理智）
新增 3 结局 `condition:'sanity<=-1'`（理智 floors 0、永不自然触发；仅 endTone 强制）。沿 outcomes 模式。
| 基调 | 性质 | 宿主事件·支 | 权重(survive:哨兵) |
|---|---|---|---|
| **同流合污·食人自保** | 地狱 | `末日食人`「装作不知，蹭一顿饱饭再走」| 1:1 |
| **弃众独生·孤鬼游荡** | 地狱 | `尸潮围城`「把同伴推出去断后，独自逃」| 1:1 |
| **以命护苗·废土微光** | 天堂 | `免疫之子`「誓死护送孩子去研究所」(护住人类解药之苗) | 4:1（稀有）|

### 8. 双死亡线 + decay 重平衡 + sim
- hp decay 1（现按月，保留——末世侵蚀）。
- `理智` decay：起 0，sim 按 36 回合定（漫长末世精神磨蚀，若疯癫死<3% 则加 1）。
- 物资靠事件每回合消耗（规则不变）。
- 跑 `npx vite-node scripts/sim-balance.ts wasteland 5000`，按 36 回合校准死亡率（力竭+疯癫+endTone 致死）、据点闸门可达性（登顶 random 稀有/survive 可达）、decay 值。

## 不做（YAGNI）
- 不加据点 flagsClear/倒退（前进式希望弧）。
- 不重写 ~80 求生事件文案（作「某月遭遇」原样可用）；只新增 4 据点闸门 + 3 身份事件 + 改造 3 哨兵宿主。
- 结局只重写 epilogue 框架 + tone 名，不大改条件轴。

## ★复用教训（务必遵守）
带 `outcomes` 的**选项本身**必须写 `effects: {}`（local.ts magOf 读 c.effects，缺则 sim 崩）；每个 outcome 分支（含纯 endTone 分支）也必须有 `effects: {}`（缺则 tsc TS2741、vitest 不报）；endTone 分支补 `reaction`。**收尾必跑 `npx tsc --noEmit`**。新增事件/身份事件前先 grep summary 防撞名。测试 import 仅取实际用到的，避免 TS6133。
